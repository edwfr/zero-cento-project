# Internationalization (i18n)

**Languages**: Italian (default) + English
**Framework**: i18next + react-i18next
**Status**: Infrastructure complete, ~270 hardcoded strings remaining (Sprint 11)

---

## Architecture

| Package                            | Purpose                                            |
| ---------------------------------- | -------------------------------------------------- |
| `i18next`                          | Core engine (detection, interpolation, namespaces) |
| `react-i18next`                    | React bindings (`useTranslation` hook)             |
| `i18next-browser-languagedetector` | Auto-detect browser language                       |

### Key Decisions

- **File-based translations** — JSON files per language in `/public/locales/{it,en}/`
- **Server + Client** — Translations available in both Server and Client Components
- **Persistence** — Language preference in cookie + localStorage
- **Fallback** — Missing translations fall back to Italian

## Usage Patterns

### Client Component

```tsx
'use client'
import { useTranslation } from 'react-i18next'

export function MyComponent() {
  const { t } = useTranslation(['trainer', 'common'])
  return <h1>{t('trainer:dashboard.title')}</h1>
}
```

### With Interpolation

```tsx
t('admin:users.confirmDelete', { count: selectedIds.size })
// → "Sei sicuro di voler eliminare 5 utenti?"
```

### Loading States

```tsx
{loading ? t('common:common.loading') : t('auth:login.submit')}
```

## API Error Keys Pattern

API routes return semantic `error.key` fields. The frontend translates using the `errors` namespace:

```typescript
// API route
return apiError('NOT_FOUND', 'Exercise not found', 404, undefined, 'exercise.notFound')

// Frontend
import { getApiErrorMessage } from '@/lib/api-error'
const msg = getApiErrorMessage(data, t('exercises.createError'), t)
// Uses data.error.key → t(key, {ns:'errors'}) if present, else fallback
```
